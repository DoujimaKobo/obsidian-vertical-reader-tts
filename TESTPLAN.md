# 試験計画書 — Vertical Reader with TTS

対象バージョン: 1.2.1
環境: Windows 11 / Obsidian デスクトップ / VOICEVOX Engine 0.25.2（http://127.0.0.1:50021）
テストVault: `同期用`（プラグイン配置・有効化済み）

このドキュメントは「何をテストするか」「どう動くのが正しいか」を定義する。
実施結果は下部の結果表に記録する。

---

## 1. 機能仕様（どう動くのが正しいか）

### A. 縦書きビュー・エディタ同期

| ID | 仕様（期待動作） |
|----|----|
| A1 | ビューを開いた瞬間、アクティブなノートの内容が縦書き（右→左）で表示される。空白にならない |
| A2 | エディタでの編集は約300ms（デバウンス）でビューに反映される |
| A3 | 同じペインで別のノートを開くと、ビューが即座にそのノートに切り替わる |
| A4 | YAMLフロントマターは縦書きビューに表示されない |
| A5 | ルビ3形式（`[漢字]{かんじ}` / `｜漢字《かんじ》` / `漢字《かんじ》`）がすべて `<ruby>` 表示される |

### B. 再生コントロール共通（OS標準・VOICEVOX両方）

| ID | 仕様 |
|----|----|
| B1 | 再生: カーソルが縦書きビュー内にあればその位置から、なければ先頭から全文を読む。**最後の文から始まったりしない** |
| B2 | 再生中は「一時停止・停止・現在位置」が表示され、「再生」ボタンは消える |
| B3 | 停止: 即座に無音になり、状態がリセットされる。次の再生は先頭（またはカーソル位置）から |
| B4 | **連打耐性**: 再生ボタンを素早く2〜5回押しても、最終的に走る再生はちょうど1つ。多重再生・無音のまま固まる・UIが再生中のまま残る、が起きない |
| B5 | 速度スライダー: ドラッグ中は数値表示のみ更新。**離した瞬間に1回だけ**再生し直す。再生していないときは再起動しない |
| B6 | 音声（話者）変更: 再生中なら新しい声で1回だけ再生し直す。ドロップダウンが空白になったりしない |

### C. OS標準（Web Speech API）固有

| ID | 仕様 |
|----|----|
| C1 | Windowsの日本語音声（Ayumi/Haruka/Ichiro/Sayaka等）が音声一覧に出る |
| C2 | 一時停止→無音。再開→**止まった位置の続き**から再生される |
| C3 | 単語境界イベントでハイライトが進む |
| C4 | 停止した直後にもう一度再生しても無音にならない（resume()キック済み） |

### D. VOICEVOX固有

| ID | 仕様 |
|----|----|
| D1 | 長文は500字以下のチャンクに分割され、プリフェッチにより途切れなく連続再生される。通知に (n/総数) が出る |
| D2 | 一時停止: 現在のチャンクで音が止まり、通知が「⏸ 一時停止中」になる。**チャンク番号が勝手に進まない** |
| D3 | 再開: 一時停止したチャンクの先頭から再合成して続きを再生（仕様: チャンク途中の秒単位からではなく、チャンク頭から） |
| D4 | 停止: 再生・プリフェッチ・進行中のHTTPリクエストがすべて破棄され無音になる |
| D5 | 話者・速度の変更は次の再生（または再起動）から反映される |
| D6 | 「ルビ優先」ONなら、ルビ注釈（読み）の方が読まれる |
| D7 | 句読点（、。！？）の休止時間設定が読み上げに反映される |
| D8 | `……` `‥` `――` `—` は内部で読点扱いになり「間」が空く。**表示テキストは変わらない** |
| D9 | 最後まで再生し終わったら、UIが自動的に再生前の状態（再生ボタン表示）に戻る |

### E. 自動スクロール

| ID | 仕様 |
|----|----|
| E1 | ONのとき、読み上げ位置に追従してスクロールする（OS=文字境界、VOICEVOX=チャンク＋チャンク内進捗の推定） |
| E2 | **一時停止中はスクロールが進まない**（読み上げ位置は止まっているのだから追従も止まる） |
| E3 | OFFのときは一切自動スクロールしない |
| E4 | 「現在位置」ボタンで、現在の読み上げ位置へ手動ジャンプできる |

### F. エンジン切替

| ID | 仕様 |
|----|----|
| F1 | パネルの「エンジン」ドロップダウンで OS標準⇔VOICEVOX を切替でき、ビューを開いたまま即反映される |
| F2 | 設定画面のドロップダウンからでも同様に切り替わる（開いているビューに反映） |
| F3 | **再生中に切り替えた場合**: 現在の再生が停止してから切り替わる。旧エンジンの音声が残って多重再生にならない |
| F4 | 切替後、自動スクロール等のチェックボックス位置・状態がUI上おかしくならない |

### G. 一括修正（エディタツール）

| ID | 仕様 |
|----|----|
| G1 | 文頭字下げ: 会話行（「『等で始まる行）以外の行頭に全角スペースを付ける。既に付いている行・フロントマターは変更しない |
| G2 | 記号後スペース: ？！の直後に全角スペースを入れる。**既存の改行は消えない**。連続記号（！？等）の間には入れない |
| G3 | すべて適用: 上記2つが順に適用される |

### H. 設定画面

| ID | 仕様 |
|----|----|
| H1 | 設定画面の文言がすべて日本語 |
| H2 | VOICEVOX: 接続テスト・話者読込・自動起動（パス指定・起動ボタン）が機能する |

## 2. 重点試験（ユーザー報告の不具合疑い）

| ID | 内容 | 関連仕様 |
|----|------|---------|
| R1 | 再生ボタン連打（素早く2回・5回、OS/VOICEVOX両方）で動作が壊れないか | B4 |
| R2 | VOICEVOX一時停止中に自動スクロールが進んでしまわないか | E2 |
| R3 | 一時停止→再開で正しい位置から続くか（OS/VOICEVOX） | C2, D2, D3 |
| R4 | 再生中のエンジン切替で音が残らないか | F3 |

## 3. 実施方法

1. **静的検査**: サブエージェントがソースコードを精査し、各試験項目の「コード上の予測結果」（Pass見込み / バグあり）と根拠を出す
2. **実機試験**: computer-use で Obsidian を直接操作し、重点試験（R1〜R4）＋主要項目を確認
3. **判定**: 期待動作 / 実測 / Pass・Fail を記録。Failは修正して再試験

## 4. 結果記録

静的検査（Opusサブエージェント）＋実機（computer-use）で実施。

### 発見・修正したバグ
| ID | 内容 | 対応 |
|----|------|------|
| R2/E2 | VOICEVOX一時停止中も自動スクロールが進み続けた（pause時 isPlaying=true のまま、checkVoicevoxScrollが isPaused を見ていなかった） | checkVoicevoxScroll と OS用スクロールに `!isPaused && autoScroll` ゲート追加 → 実機で**PASS**（7秒待って位置不変） |
| E3 | 自動スクロールのチェック状態が VerticalReader に伝わっていなかった（TTSControlsローカル変数） | `ttsState.autoScroll` ストアに昇格し両者で共有 |
| R1/B4 | 再生ボタン連打で複数の stop/speak が競合しうる | handlePlay に `isStarting` ガード＋即 `ttsState.play()`（再生ボタン即無効化） |
| UI | 停止時と再生時で再生ボタンの位置がずれ、連打が別ボタン（一時停止等）に当たる | トランスポート4ボタンを常時表示・無効化で切替（位置固定）＋無効スタイル追加 |

### 検証済み（実機・コンソールログで確認）
| ID | 結果 |
|----|------|
| D2 一時停止 | 「一時停止中」表示、コンソール `Halted after playback completion` でチャンク進行停止を確認 → **PASS** |
| D3 再開 | `Starting seamless playback from chunk N` で停止チャンクから再合成・継続再生 → **PASS**（仕様: 現チャンク先頭から） |
| D7 句読点休止 | コンソール `Applied pauses - comma:0.3 period:0.5...` → **PASS** |
| D8 …／― | normalizePauseSymbols で読点化、両エンジン適用 → **PASS**（コード＋再生確認） |
| B1 カーソル再生 | カーソル位置から再生、末尾なら全文フォールバック → **PASS** |
| B2/B3 | 再生/停止で状態遷移正常 → **PASS** |
| A1 表示 | 縦書き即描画。※開いた直後の同期前一瞬だけ content 空で「テキストがありません」になりうる（軽微、要observ） |
